# 🚀 Gemini Portfolio Site

Персональний сайт-портфоліо, розроблений з використанням сучасних методологій розробки (DevOps), автоматизованого тестування та версіонування баз даних.

## 🛠 Технологічний стек
Проєкт демонструє володіння наступними інструментами:
* **Frontend:** HTML5, CSS3, JavaScript (інтеграція з Gemini API).
* **Контейнеризація:** [Docker](https://www.docker.com/) (оптимізовані образи на базі Nginx Alpine).
* **CI/CD:** GitHub Actions (автоматичний лінтинг, збірка та тестування).
* **Тестування:** [k6](https://k6.io/) (навантажувальне тестування продуктивності).
* **Автоматизація:** [Renovate Bot](https://www.whitesourcesoftware.com/free-developer-tools/renovate/) (керування залежностями).
* **Бази даних:** SQLite та [Liquibase](https://www.liquibase.org/) (версіонування схеми БД).

---

## 📈 Еволюція проєкту

### 🟢 Етап 1: Розробка та Контейнеризація
Створення структури сайту та пакування застосунку у Docker-контейнер для забезпечення ідентичності середовища розробки та розгортання.

### 🟡 Етап 2: CI/CD та Автоматизація
Налаштування автоматичного пайплайну в **GitHub Actions**:
* **Linter:** Статичний аналіз коду на наявність помилок.
* **Docker Build:** Автоматична збірка образу при кожному push.
* **Renovate:** Автоматичне відстеження та оновлення версій бібліотек та Docker-образів.

### 🔴 Етап 3: Performance Testing (k6)
Інтеграція тестів продуктивності у CI/CD. Кожен комміт перевіряється на відповідність порогам (thresholds):
* 95% запитів мають оброблятися швидше ніж за 500мс.
* Статус успішних відповідей сервера має бути 100%.

### 🔵 Етап 4: Версіонування БД (Liquibase) — РГР
Додано систему керування змінами бази даних. Використання Liquibase дозволяє відстежувати історію структури таблиць у форматі XML та за потреби робити відкат (rollback).

## 🏗 Керування базою даних (Liquibase)

У проєкті використовується **SQLite** як легка портативна база даних.

### Основні файли:
* `db/changelog.xml` — журнал усіх змін структури БД.
* `db/my_portfolio.db` — файл бази даних.
* `liquibase.properties` — конфігурація підключення.
